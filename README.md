# Ateka Automation \& Order Processing Portal ⚙️

מערכת אוטומטית לקליטה, פיצול ופענוח של הזמנות רכש מלקוחות בפורמט Excel, CSV ו-PDF דיגיטלי.

המערכת מחלצת מק"טים וכמויות, מצליבה אותם מול קטלוג החברה, מתקנת פורמטים, ומייצרת קובץ Excel מוכן לעבודה עם שתי לשוניות נפרדות:

1. **העתקה לפריוריטי** - מבנה מותאם להעתקה והדבקה למסך אב בפריוריטי.
2. **טעינה לפורטל** - מבנה רגיל לטעינה לפורטל ספקים.

המערכת רצה ב-Railway בשני שירותים נפרדים: Web App ו-Email Worker.

\---

## 🌟 תכונות מרכזיות - גרסת Production

* **מנגנון פיצול PDF חכם (`pdf\\\_router.py`)**  
מזהה אוטומטית קבצי PDF מרוכזים המכילים מספר הזמנות רכש, בעיקר לפי תבנית `PO`, ומפצל אותם בזיכרון להזמנות נפרדות.
* **עבודה דו-ערוצית במקביל**

  * **Web App ב-Streamlit (`app.py`)** - העלאה ידנית של Excel / CSV / PDF וקבלת קובץ מוכן להורדה.
  * **Email Worker (`email\\\_worker.py`)** - מאזין לתיבת מייל, מזהה קבצים מצורפים, מעבד אותם, ומשיב לשולח עם קובץ Excel מוכן ועם PDF מקור חתוך לפי הזמנה.
* **PostgreSQL במקום חיפוש ישיר בקובץ CSV**  
קטלוג החברה `PB.csv` נטען לטבלת `catalog` במסד PostgreSQL.  
ב-Railway החיבור מתבצע דרך משתנה הסביבה `DATABASE\\\_URL`.  
במחשב מקומי, אם `DATABASE\\\_URL` לא מוגדר, המערכת מתחברת אוטומטית ל-PostgreSQL מקומי ב-Docker:

```text
  postgresql://postgres:1234@localhost:5432/postgres
  ```

* **טעינה אוטומטית של הקטלוג**  
המערכת בודקת האם טבלת `catalog` קיימת. אם הטבלה לא קיימת או לא נגישה, היא טוענת את `PB.csv` ויוצרת את הטבלה מחדש.
* **Matching חכם למק"טים**

  * חיפוש לפי `vendor\\\_sku`
  * חיפוש לפי `vendor\\\_sku` אחרי ניקוי רווחים ומקפים
  * חיפוש לפי `ateka\\\_sku`
  * השלמת מק"ט אטקה ל-9 ספרות עם אפסים מובילים
  * הסרת תחיליות מיותרות כגון `AT-`, `AT\\\_`, `AT`, `A-`
  * התעלמות ממספרי פרויקט שמתחילים ב-`PR`
  * סינון שורות רעש כגון `510050`
* **קובץ Excel מוכן לעבודה**

  * שתי לשוניות: `העתקה לפריוריטי` ו-`טעינה לפורטל`
  * פונט Tahoma
  * גודל פונט 14
  * יישור למרכז בתוך התאים
  * צבע ירוק בהיר לשורות תקינות
  * צבע כתום/צהוב לשורות עם הערות מערכת
  * מסגרות לכל הטבלה
  * Freeze panes לשורת הכותרת
  * Auto filter
  * רוחבי עמודות מוגדרים מראש
  * כל הגדרות העיצוב מרוכזות בתחילת `excel\\\_handler.py` כדי שיהיה קל לשנות בעתיד
* **ניהול שגיאות בתוך האקסל**
כל שורה בעייתית מסומנת ומקבלת הערה בעמודת `הערות מערכת`, לדוגמה:

  * מק"ט לא מוכר
  * חסרה כמות תקינה
* **התראות Telegram**
לאחר פיענוח קובץ או הזמנה, נשלחת התראה עם:

  * מספר הזמנה / שם קובץ
  * מספר שורות שפוענחו
  * מספר שורות שדורשות בדיקה
* **אבטחת שימוש**

  * ה-Email Worker מטפל רק בשולחים מדומיינים מורשים.
  * פענוח PDF באתר דורש סיסמת הרשאה.
  * מסמכים סרוקים / תמונות נחסמים כדי למנוע טעויות באספקה.

## 📂 מבנה קבצי הפרויקט

* `app.py`  
ממשק Streamlit להעלאה ידנית של קבצי Excel, CSV ו-PDF.  
מאפשר למשתמש לקבל קובץ Excel מוכן להורדה.
* `email\\\_worker.py`  
שירות רקע שרץ ב-Railway, מאזין למיילים חדשים דרך IMAP, מעבד קבצים מצורפים, ושולח מייל תשובה דרך SMTP.
* `db\\\_handler.py`  
שכבת החיבור ל-PostgreSQL.  
אחראית על:

  * קריאת `DATABASE\\\_URL`
  * fallback ל-DB מקומי
  * טעינת `PB.csv` לטבלת `catalog`
  * חיפוש מק"טים בטבלה
  * השלמת מק"ט אטקה ל-9 ספרות
* `excel\\\_handler.py`  
שכבת יצירת האקסלים.  
אחראית על:

  * הצלבת מק"טים מול DB
  * יצירת לשונית `העתקה לפריוריטי`
  * יצירת לשונית `טעינה לפורטל`
  * עיצוב הקובץ
  * סימון שורות תקינות ובעייתיות
* `pdf\\\_router.py`  
מזהה הזמנות מרובות בתוך PDF ומפצל אותן לקבצי PDF נפרדים בזיכרון.
* `pdf\\\_handler.py`  
מנוע AI לפענוח PDF בעזרת OpenAI API.  
מחלץ שורות הזמנה, מק"טים וכמויות.
* `telegram\\\_handler.py`  
שליחת התראות Telegram על שימוש במערכת.
* `PB.csv`  
קטלוג החברה.  
משמש לטעינת טבלת `catalog` ב-PostgreSQL.
* `requirements.txt`  
רשימת ספריות Python הנדרשות להרצת המערכת.
* `Procfile`  
קובץ פקודות הפעלה ל-Railway / סביבת deployment.

## 🗄️ PostgreSQL וטעינת קטלוג

המערכת משתמשת ב-PostgreSQL כדי לבצע חיפוש מהיר ואמין מול קטלוג החברה.

### טבלת `catalog`

הטבלה נוצרת מתוך `PB.csv` וכוללת את העמודות:

```text
ateka\\\_sku
vendor\\\_sku
hebrew\\\_desc
english\\\_desc
```

### לוגיקת טעינה

ב-`db\\\_handler.py` קיימת פונקציה שטוענת את הקטלוג מתוך `PB.csv`:

```python
init\\\_catalog\\\_db("PB.csv")
```

בנוסף קיימת פונקציית בדיקה וטעינה אוטומטית:

```python
ensure\\\_catalog\\\_db("PB.csv")
```

הלוגיקה המומלצת:

```text
אם טבלת catalog קיימת - לא טוענים מחדש.
אם טבלת catalog לא קיימת - טוענים את PB.csv ויוצרים אותה.
```

### חיבור מקומי מול Railway

ב-`db\\\_handler.py` החיבור מוגדר כך:

```python
DB\\\_URL = os.getenv("DATABASE\\\_URL", "postgresql://postgres:1234@localhost:5432/postgres")
```

המשמעות:

```text
Railway  -> משתמש ב-DATABASE\\\_URL של Railway
מחשב מקומי -> משתמש ב-PostgreSQL Docker מקומי
```

### הרצת PostgreSQL מקומי ב-Docker

```powershell
docker run --name ateka-db -e POSTGRES\\\_PASSWORD=1234 -p 5432:5432 -d postgres
```

עצירה זמנית:

```powershell
docker stop ateka-db
```

הפעלה מחדש:

```powershell
docker start ateka-db
```

## 📧 Workflow של Email Worker

1. השירות מאזין למיילים חדשים בתיבת הבוט.
2. הוא מטפל רק במיילים שלא נקראו.
3. הוא בודק שהשולח שייך לדומיין מורשה.
4. הוא מזהה קבצים מצורפים מסוג:

   * PDF
   * XLSX
   * XLS
   * CSV
5. אם הקובץ הוא PDF:

   * מפצל להזמנות נפרדות
   * מפענח כל הזמנה עם OpenAI
   * מייצר Excel לכל הזמנה
   * מצרף גם את ה-PDF המקורי החתוך של אותה הזמנה
6. אם הקובץ הוא Excel / CSV:

   * קורא את המק"טים והכמויות
   * מצליב מול DB
   * מייצר Excel מוכן
7. שולח מייל תשובה למשתמש.
8. מסמן את המייל כנקרא.
